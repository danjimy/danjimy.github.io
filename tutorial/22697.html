单机魔域修改技术大合集

发表于2024-06-24|更新于2024-05-16|tutorial
|阅读量:

一，合BB所需副BB评分如何设定为最高只需19星XO

打开数据表，找到cq_eudemon_rbn_rqr这张表。然后找到 sacrifice_starlev 这一列。把高于1900的数字改为1900，那么你BB不管多少星。所需要的副BB只需要19星就可以合了。

二，如何让你的BB出生即为高星

打开数据表，找到cq_grade这张表，然后，拉到后面，找到列名为rarity 的竖排，把这里的数字改掉。比如你想要100*的就改10000就可以了。不过不要改的太高，以免不能召唤。

三，BB战改法或法改战

在cq_itemtype下面,找到要改的BB的ID 找到able_mask这栏,把这栏里的改掉好了

7代表可以骑和合体 （非骑宠，改过后没什么用）

3代表合体 （法改战，要拥有战士属性，需要改cq_grade属性）

2就是不可以合体 （战改法，要拥有法师属性，需要改cq_grade属性,同时在cq_action配相关技能）

四，修改合BB的速度（即每次加的评分）

打开数据库，找到名为cq_eudemon_rbn_rqr的表，你会看到列 min 和 max这两列对应都有数字。上面的一般是4000+ 越到下面数字越小你的可能是1或2. 我们如果想要让BB合得快，一般只要把 max里的数字加大即可让你的BB每次评分升高。具体加多少，请自己去试.

五，如何修改合BB不用相同属性

打开cq_eudemon_rbn_rqr表，找到列名为 sacrifice_dmg_type 这里 0代表不需要相同属性， 1为需要相同属性。把1改成0，那么就不受限制了。

六，如何修改BB合星时成功与失败的机率

打开数据库，cq_eudemon_rbn_rqr表，找到列名为 suc_percent 把这里的数字改成100，那合BB不会失败，一般最低改成75就很容易失败了！

七，如何修改天天军团战

打开数据库，找到cq_config这张表。
然后找到ID 191（191 7000 5213000 5233000 21150 0 0 0

把这第三个和第四个的第一个数字 5 改成1就是星期一打，改成3就是星期三打了。

可以从1到7一样的改一个。然后把这表复制出来当成补丁，每天重启服务器时，复制当天日期的替换原来
这个现在都是由MSG控制了，无需在这里修改

八，修改VIP批量孵化时间

找到cq_config这张表
找到type为3101 到3107的行，ID不说了，有可能你的不同！

3101就是1级VIP。3107就是7级VIP了。
然后把对应的data1 里面的数字改掉（这里1就相当于一秒）改成你想要的时间就OK！
不过现在都是gm工具直接修改了

九，修改技能经验及等级

用Navicat这个工具,打开cq_magictype这个表 找到need_exp和need_level这两列 need_exp指升级需要的经验；觉得升的慢可以改的低点。need_level指技能等级提升需要的级别

修改怪物和BB或者部分含攻击性的NPC， 打开cq_monstertype这张表，找到你想修改的怪物的ID，在第四列名为lookface的行里面，

10711 10791就是第一次进化后的外型

功能NPC外型的修改，我们打开cq_npc表，选择你要修改的NPC的ID那行，在第五列名为lookface这里，同样修改成你想要他变的外型代码就行了。
修改之前，必须知道你想要改过后外型的代码！

十九，如何实现批量修改暴率

首先用Navicat 打开数据库,然后找到 cq_dropitemrule 表, 这个就是怪物暴率+暴物品的表

点查询,新建查询 输入 update cq_dropitemrule set chance=200000 where chance=500000 点执行即可

暴率为chance这列

200000为要改的暴率 数值的表示: 20万较低 50万中等 100万较高

500000为当前的暴率

item0-item14 都是所暴物品的ID,关于ID代表的物品可以在cq_itemtype表内查询对应ID了解到, 可以自己修改为想要掉的物品 item编号越低越容易出!

下面我们再来看看批量修改所暴物品的范例查询命令:

update cq_dropitemrule set item14=要设置掉的物品ID where item14=当前这列的物品IDitem14为最后一个物品列 可以自己修改item0-item14

好了,去试试吧,不过有两点需要注意;

在修改前先备份一下数据库,什么,你不知道怎么备份,那就说得简单通俗一点,找到sql server里mysql文件夹下data目录里的my文件夹进行备份(有account文件夹的话也备份一下).

修改完要记得重新启动服务端程序才能生效.

二十六，增加新BB要注意的地方

很多人有了C3和texture里面的DDS文件,却不知道增加新BB需要从哪里入手,今天我就简单说下需要相应在数据库和客户端增加的地方

首先来看数据库,主要需要增加以下表中的数据:

cq_action(里面增加打开幻兽蛋的部分)

cq_monstertype(里面增加怪物类型数据)

cq_itemtype(物品,增加6条BB数据,一个礼包数据,一个BB蛋数据)

cq_grade(增加BB的基本数据,如生命,攻击防御,成长,罕见度等)

cq_eudmon_rbn_type(增加BB幻化类型数据)

cq_dyna_rank_type(增加BB排名数据)

cq_totem_pole_type(增加BB图腾数据)

cq_talentgain(增加BB获得天赋数据)

cq_goods(增加商店出售BB数据,此数据不加可用物价补丁软件调,无所谓)

再看客户端:

先看ANI目录下:

ItemMiniIcon(BB蛋的显示配置,一个BB8个数据配置,蛋一个,礼包一个,外加BB的6条数据配置)

PetFace(BB的头像,游戏上方的显示.一个BB6条数据配置,128的是背包里面的显示,round的是游戏上方的头像)

MapItemIcon(这个是物品掉地上的图)

再看INI文件里面哪些文件需要配置:

itemtype.dat(一个BB需要增加8个数据,增加6条BB数据,一个礼包数据,一个BB蛋数据)

shop.dat(商店的,可不加,GM口令刷或者物价软件自己调)

3dmotion.ini(控制BB动作的)

3dobject.ini

3DSimpleObj.ini

3DSimpleRole.ini

3DTexture.ini

armor.ini

主要是这几个,有些东西用几句话说不清楚,反正大家打开了自己去参考其他BB添加就是了.

另外还有就个文件:

Eudbook.ini

EudbookDes.ini

PetDescribe.ini

Totempole.ini

SysTotemPillar.ini

这几个文件是关于BB分类,BB介绍,描述,图腾和图腾分布情况的,自己一样参考其他BB去改吧.

二十七，使用GM工具加了暴的东西，打怪时很难出来

我们使用GM工具时一般他修改的是cq_dropitemrule这张表里的东西，但是和暴率有关的另一处地方在cq_monstertype里面. 我们修改了暴率，同时也需要修改cq_monstertype的东西才行。

先看cq_dropitemrulecq_dropitemrule 里的group为你要设定暴东西的怪物ID（要和cq_monstertype里的ID对应）

chance列即为暴率 后面的item0到item14为暴的物品（要暴什么填上物品的代码就行）

然后我们看cq_monstertype

cq_monstertype 这个是怪物掉东西的暴率

drop_money_min 爆钱最少数值(就是爆出来那一堆里单个钱的数量)

drop_money_max 爆钱最少数值(就是爆出来那一堆里单个钱的数量)

drop_money_chance 掉钱几率

drop_item_chance 掉道具概率

explode_item_chance1 爆道具概率1

explode_item_chance2 爆道具概率2

explode_item_chance3 爆道具概率3

drop_item_rule 掉道具规律

修改这上面对应的数据就可以控制你要暴的物品了。

打开数据库，找到名为cq_eudemon_rbn_rqr的表，你会看到列 min 和 max
这两列对应都有数字。上面的一般是4000+ 越到下面数字越小 你的可能是1 或 2
我们如果想要让BB合得快，一般只要把 max里的数字加大即可让你的BB每次评分升高。具体加多少，请自己去试.

三十七， NPC和野怪修改

  1. NPC修改的部分在“my”里“cq_npc”。要改的东西其实不多。
    id是NPC列表的顺序，name是NPC名称，lookface是NPC外观，task0是npc要执行的行为
    cellX和celly是NPC的坐标，mapid是NPC所在城市地图代码。其他不用改。
  2. 野怪修改在“cq_monstertype”，name是名称，lookface是野怪外观代码，life是生命血量，attack-max、min是野怪攻击力，dodge是野怪闪避，defence是野怪防御，action是野怪的执行行为。野怪也包括自己的携带幻兽，改法是一样的。

三十八，物品的修改

物品在“cq_itemtype”，魔域中的所有物品都在这里，特别是装备的属性修改也在这，修改装备之后，还要相应的在客户端补丁里面也要修改，在ini文件夹里，找到itemtype，这里是修改物品的显示，就是说你在数据库中修改了之后，只是改掉了实际的属性，但是游戏中你是看不到你改过的属性的，还要在客户端补丁中修改以后，两者才同步。

三十九技能属性修改

在“cq_magictype”里，use-mp是技能魔法消耗，use-ep是技能体力消耗，need-exp是技能升级需要的经验，等等，其他的没什么要改的

四十，幻兽合一次加的评分修改

在“cq_eudemon_rbn_rqr”里，需要改的是max这一列，点右键，排序，升幂排序，可以看到2,3等等的评分，这是用幸运草之后最大评分，改成10就是合一次加10分，以此类推。
